Glossary

Assets: the files stored in jdvault. An asset includes images, video, audio clip, logos, illustrations, animations, presentations, spreadsheets, word documents, electronic        mails, and a multitude of other digital formats.

Asset Details: The metadata content displayed in the right pane of the asset preview.

Additional files: Related to an asset file.If you click on a single file, it will also show the additional asset files. no preview is provided.

Asset Codes: Autogenerated series of numbers and letters (codes) assigned to assets for identification and tracking purposes.

Activity Logs: Contains a news feed format list of notable changes made to the asset metadata and also of asset created, viewed and downloaded details.

Batch Upload: The ability to upload, a group of assets simultaneously

Batch Edit: The ability to select a group of assets to make the same changes to simultaneously.

Compression: Compression is used during download of digital media file formats to reduce the amount of storage capacity required to hold files on a storage medium and to enable faster transmission over networks. The process of compressing data uses one or more algorithms that identifies potential inefficiencies in terms of how the file is stored. The algorithm needs to both compress and decompress the file so it can be saved and opened again later

Custom Attributes: defined relevant attributes that describes the asset

Description: Applying a well-designed and descriptive structure of keywords to the asset

Filters: a combination of search terms that is relatively specific for narrowing down search and finding results quickly. Controls like drop-down menus, checkboxes are used to refine their search criteria.

File format: File format is a term to describe the type of each digital file. Some file formats are product-specific and will be named after the software or hardware used to create them. Others are more generic and may be created by a wide range of tools.

Keyword: A keyword is applied to a specific asset cataloging activity where words, phrases or terminology (or 'keywords') are attributed to assets as metadata.

Metadata: is referred to as 'data about data' and refers to descriptive information applied to assets to support a task or activity. Mostly it helps to help users locate assets in searches. To help find suitable media, assets will generally have short descriptions or titles added to them as a basic minimum, although it is more common to add much more descriptive detail to help users to locate what they are looking for. As well as search metadata, workflow and business process information may also be added to determine what procedures are followed when users want to download assets.

Note: provides information that is relevant to a topic but that may not apply to all

Permission based Access: Individual users are assigned user levels which control what features of jd vault they can access.

FTP Server: File Transfer Protocol (FTP) is a standard network protocol used for the transfer of asset files between the user's system and server over a network.

Reports: Documents containing information in a tabular form prepared based on a required basis referring to a specified time period.

Saved Filters: Selected filter selection is saved to save time finding and applying the same filters to find the same set of assets.

Search: search tools are used to locate a selection of assets within the wider repository is probably the biggest single reason users decide to invest in DAM. Search facilities can cover a variety of different capabilities, including keywords, facets and filters. These are not mutually exclusive and as well as search, users may also employ controlled vocabularies to help them find suitable assets.

System attributes: Standard attributes that are applied to all assets and include an autogenerated asset code, the asset name or title, description and tags.

Tags: Labels or terms attached to assets for the purpose of identification of the asset.

Thumbnail: Small versions of the asset that gives sufficient information about it.

UI: User Interface (or 'UI') is a generic IT term relating to the method by which a user operates computer software. User interfaces typically are either graphical or command oriented (i.e. written instructions).

Uploading:  the process of transferring digital assets to the jd vault Digital Asset Management system.  Uploading refers to the process of transferring a digital asset from a user's computer to the Digital Asset Management system, jd vault. In most cases these days, it implies transferring the asset over either the internet or a corporate intranet.

Zip file: ZIP is a popular file compression format used to reduce the storage space consumed by digital files.
